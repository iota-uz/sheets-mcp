# Import bank statement

Разбор банковской выписки в лист Google Sheets. Используется для расчётного счёта (Kapital UZS).

## Когда применять

- "обработай отчёт", "обработай выписку", "импортируй выписку"
- Пользователь указал .xlsx файл в `reports/`
- Упомянул "xlsx" или "банковскую выписку"

## Шаг 1 — прочитать xlsx

```bash
node -e "import('./mcp/excel.mjs').then(m => { const r = m.readBankStatement('reports/<файл>.xlsx'); console.log(JSON.stringify(r, null, 2)); });"
```

Это вернёт массив строк с полями: `doc_date`, `debit`, `credit`, `account_name`, `description`, и т.д.

Уточни у пользователя период (обычно один календарный месяц, формат "март 2026" → фильтр `/\.03\.2026$/` по doc_date). Если пользователь не сказал период — спроси.

## Шаг 2 — узнать структуру таблицы

```
sheets_info
```

Получишь список листов. Определи целевые по структуре (через `sheets_info sheet=<имя>` — покажет первые 3 строки):
- Лист с колонками "Дата", "Категория", "Сумма", "Курс", "Сотрудник", "Описание" → это лист расходов
- Лист с "Итерация", "Проект", "Сумма", "Категория", "Входящий счёт" → лист платежей
- Лист с "Исходящий счет", "Входящий счет", "Тип" → лист ддс
- Лист с "Проект", "ID", "#", "Оплачено", "Остаток" → лист итераций

**Не хардкодь имена.** Определяй по заголовкам.

## Шаг 3 — классифицировать каждую строку

Для каждой строки xlsx:

### Определение типа

| Условие | Тип | Куда пишем |
|---------|-----|------------|
| `credit > 0` | Поступление | лист платежей + создать итерацию |
| `debit > 0` + описание содержит "фин займ"/"фин помощь" или контрагент содержит "Транзитный счёт ... Самарканд" | Внутренний перевод | лист ддс |
| `debit > 0` иначе | Расход | лист расходов |

### Расходы — определение категории (по смыслу, не regex)

Читай описание и контрагента, используй здравый смысл:
- Контрагент `BEFORMED`, слова "ЗП", "зарплат", "премиальн", "премия", "неиспользованный отпуск" → **ФОТ**
- "обслуживание пассивных счетов", "пенсия", "НДФЛ", узбекские налоговые термины ("жисмоний шахслар даромад", "?ўшилган ?иймат", "импорт услуг", "чет эл юридик") → **Налоги**
- "тариф", "CORPORATE", "SWIFT", "комиссия инобанка", "за документ S=" → **Комиссия банка**
- "реклама", "размещение", "публикация", "пакет сайт", "партнерство ассоциация", "маркетинг" → **Маркетинг и реклама**
- "аренда помещения", "аренда офис" → **Аренда офис**
- "A3 LIST", "ROLLAB", "стенд", "полиграф" → **Офисные припасы**
- "бухгалтерские услуги", "аудит" → **Наемные рабоники вне штата**
- Ничего не подошло → **Другое**

Если категория "ФОТ" — извлеки имя сотрудника из описания (обычно в конце, перед номером карты или "данные карты").

### Платежи — определение проекта (по смыслу + сумма)

- Описание содержит "АИС GRANITE" или "разработка и внедрение автоматизированной системы" → **Granite** / `Разработка ПО под ключ`
- Описание содержит "консалтинговые услуги по цифровизации" **И сумма = 101 000 000** → **Takhir** / `Консалтинг и продажа часов`
- Описание содержит "цифровое курирование" или ("консалтинг" + сумма > 300M) → **Reklama EAI** / `Другие источники дохода`
- Описание содержит "техническая поддержка, сопровождение веб" или ("консалтинг" + средняя сумма) → **EAI_Website** / `Поддержка и мелкие доработки`

## Шаг 4 — показать dry-run пользователю

Перед записью **ВСЕГДА** покажи пользователю итоговую классификацию в виде таблицы:
- Сколько расходов (и разбивка по категориям)
- Сколько платежей (и по каким проектам) + **явно упомяни что будут созданы N новых итераций**
- Сколько ддс

Жди подтверждение ("да"/"пиши"/"ок"). Не записывай пока не подтвердили.

Если пользователь сказал `--dry-run` — просто показать и выйти.

## Шаг 5 — запись (порядок важен!)

### 5.1. ПЕРВЫМ ДЕЛОМ — итерации для каждого платежа

**Это обязательно.** Платёж без итерации не имеет смысла — лист платежей ссылается на ID итерации.

Для каждого платежа:
1. `sheets_read` листа итераций (колонки A:C) → собери все строки
2. Отфильтруй по проекту, найди max номер (колонка C) для этого проекта
3. Новый ID = `<Проект>-<max+1>`. Пример: последний `Granite-15` → новый `Granite-16`
4. Запиши в лист итераций через **smart-append**:

```bash
node scripts/smart-append.mjs --sheet=<имя_листа_итераций> --values='{"Проект":"Granite","ID":"Granite-16","#":16,"Оплачено":316000000,"Остаток":316000000,"Валюта":"SOM"}'
```

Проверь что smart-append вернул `appended: true` (не `skipped: true` — значит дубль, пропускай эту итерацию).

### 5.2. ПОТОМ — платёж со ссылкой на свежесозданную итерацию

```bash
node scripts/smart-append.mjs --sheet=<имя_листа_платежей> --values='{"Итерация":"Granite-16","Проект":"Granite","Сумма":316000000,"Категория":"Разработка ПО под ключ","Валюта":"SOM","Дата":"10.03.2026","Входящий счёт":"Kapital UZS (р/с)","Комментарий":"На р/с"}'
```

smart-append сам заполнит курс, сумму $ и расчётный период.

### 5.3. Расходы и ддс

Для каждого расхода/ддс вызови smart-append на нужный лист. Счёт для выписок с р/с всегда `Kapital UZS (р/с)` (для расходов) или дефолты в ддс (см. лист "Счета" для точных названий).

## Шаг 6 — отчёт пользователю

Когда всё записано — короткий отчёт:
- Записано платежей: N (итерации: <список ID>)
- Записано расходов: N
- Записано ддс: N
- Пропущено дублей: N (если были)
- Ошибок: N (если были — список)

## Чего НЕ делать

- ❌ **Не пиши платёж до итерации.** Сначала итерация, потом платёж.
- ❌ **Не записывай без dry-run подтверждения** от пользователя (если не указан `--dry-run`).
- ❌ **Не хардкодь имена листов** — определяй через `sheets_info` + заголовки.
- ❌ **Не пиши сразу всё массово через bash loop** — это оптом, пользователь теряет контроль. Лучше ограничить показом dry-run и дать подтверждение.
- ❌ **Не игнорируй "skipped: true"** от smart-append — это дубль, значит уже записано, пропускай с предупреждением.

## Детали xlsx-формата

Kapital Bank exportит колонки: "№ пп", "Дата документа", "Дата обработки", "№ док", "Наименование счёта" (контрагент), "ИНН", "№ счёта", "МФО", "Обороты по дебету", "Обороты по кредиту", "Назначение платежа".

Парсер `mcp/excel.mjs` находит заголовок автоматически. Если xlsx из другого банка — формат может отличаться; тогда скажи пользователю что требуется доработка парсера.

## Счёт для расходов

Для расходов из выписки с р/с — счёт всегда точное название как в листе "Счета", обычно `Kapital UZS (р/с)`. Проверь точное название через `sheets_read sheet="Счета"`.
