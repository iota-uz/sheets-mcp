# Справка для skill'а balance-check

Подробности по терминам, паттернам и структуре данных. Подгружай когда нужны детали, не входящие в SKILL.md.

## Орфан (orphan)

Транзакция, у которой в колонке "счёт" указано название, которого **нет в листе "Счета"**.

**Пример:** в Счета есть `Uzum Visa UZS`, а в расходах кто-то записал `Uzum UZS`. Эти деньги не учитываются в потоке по конкретному счёту и могут создать искусственное расхождение.

**Причины:**
- Опечатка при ручной записи
- Сокращение названия ("Uzum UZS" вместо полного "Uzum Visa UZS")
- Разные раскладки (`Kapital UZS (р/с)` с кириллицей vs `Kapital UZS (p/c)` с латиницей)

**Как чинить:** исправить ячейку в транзакционном листе через `sheets_update` — название должно **ровно** совпадать с ячейкой A в листе "Счета".

## Паттерны подозрительных транзакций

Поле `suspects[].reason` из JSON-вывода скрипта:

### `exact_match`
Сумма транзакции в долларах равна модулю разницы.
- **Интерпретация:** эта запись либо лишняя, либо её нет в реальности, либо наоборот — не хватает одной такой же.
- **Действие:** сверить эту строку с выпиской / чеком.

### `doubled`
Сумма × 2 = разница.
- **Интерпретация:** возможно эта запись попала в таблицу дважды.
- **Действие:** поискать другую строку с той же датой/суммой/контрагентом.

### `halved`
Сумма / 2 = разница.
- **Интерпретация:** возможно в таблицу записали двойную сумму (например, записали платёж клиента + бонус одной строкой вместо двух).
- **Действие:** проверить что сумма не склеена из двух.

### `currency_error`
Сумма в SOM близка к модулю разницы (разница в долларах).
- **Интерпретация:** скорее всего при записи перепутали валюту (написали SOM, имея в виду USD, или наоборот).
- **Действие:** проверить колонку F в транзакционном листе.

### `decimal_shift`
Сумма отличается от разницы ровно в 10 раз.
- **Интерпретация:** лишний или пропущенный ноль при вводе.
- **Действие:** перепроверить цифру с выпиской/чеком.

## Расчётный период

Формат: `01.MM.YYYY` (всегда первое число месяца).

Пример: транзакция 15 марта 2026 → расчётный период `01.03.2026`.

Присваивается автоматически при записи через `scripts/smart-append.mjs` (auto-fill из колонки "Дата"). В сверке используется как фильтр "за какой месяц смотреть".

## Направление разницы

Разница = Фактический баланс − Баланс согласно таблице (ячейка G21 в Счета).

- **Положительная** (Δ > 0): реальности больше чем в таблице → не хватает записей о доходах → ищем пропущенный **платёж** (клиент заплатил, не записали).
- **Отрицательная** (Δ < 0): реальности меньше чем в таблице → не хватает записей о расходах → ищем пропущенный **расход или ддс** (деньги ушли, не записали).
- **Около нуля** (−$2000 < Δ < +$7000): норма для этой таблицы, не копаем.

## Точные названия счетов из листа "Счета"

**Важно:** при записи транзакций название счёта должно **ровно совпадать** с колонкой A в листе "Счета" (регистр, пробелы, скобки). Иначе → орфан.

Актуальный список (проверяй через `sheets_read` с sheet="Счета"):

- `Kapital UZS (р/с)` — расчётный счёт UZS
- `Kapital USD (р/с)` — расчётный счёт USD
- `Tenge Humo`
- `UzCard (Kapital)`
- `Uzum Visa UZS` — **НЕ** `Uzum UZS`
- `Tenge Mastercard`
- `Uzum Visa USD`
- `Наличные USD`
- `Наличные UZS`
- `USDT TG Wallet`
- `USDT Binance`

## Структура вывода скрипта (JSON)

```js
{
  period: "01.04.2026",
  summary: {
    tableBalanceUsd,       // Баланс согласно таблице
    actualBalanceUsd,      // Фактический баланс
    differenceUsd,         // Разница (Фактический − Таблица)
    absDifferenceUsd,      // |Δ|
  },
  perAccount: [{           // по каждому счёту из Счета
    name, currency,
    actualAmount, actualUsd,
    incomingUsd, outgoingUsd,
    transfersInUsd, transfersOutUsd,
    netFlowUsd,            // нетто-поток за всё время
    txCount,
  }],
  orphans: [{              // счета, встречающиеся в транзакциях, но отсутствующие в Счета
    account,
    incomingUsd, outgoingUsd,
    transfersInUsd, transfersOutUsd,
    txCount,
  }],
  inPeriodCounts: { expenses, payments, transfers },
  suspects: [{             // подозрительные транзакции (только если |Δ| > tolerance)
    sheet, row, reason, note,
    usd, raw, currency, description,
  }],
  topLargestInPeriod: [{   // топ-5 крупнейших за период для глазного контроля
    sheet, row, usd, raw, currency, description,
  }],
}
```
